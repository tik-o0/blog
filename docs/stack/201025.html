# jamstack

サーバーに依存しない -> サーバーレスである
静的サイトで動的なコンテンツを扱うための仕組み
ユーザーによって表示が可変しないようなコンテンツに向いている
（動的な処理で都度ビルドが発生するので、負荷が高くなる）

# 構成要素

  • JavaScript
  • APIs

上記で事前にデータ等を取得し、ビルドする

  • Markup

# APIs

採用したのは microCMS
ヘッドレスCMS -> 端的に言えばViewをもたないCMS
